Contributing to Connecdo
First off, thank you for considering contributing to Connecdo! 🙌

We’re excited to have you help build a platform that empowers people to turn “I wish this existed” moments into real solutions.

How Can You Contribute?

Report bugs: Found something not working right? Please open an issue.

Request features: Have an idea to improve Connecdo? We want to hear it!

Submit code: Fix bugs, add features, or improve documentation.

Improve docs: Better README, tutorials, or examples are always welcome.

Help others: Answer questions or guide newcomers in discussions.

Getting Started

Fork the repo
Click the “Fork” button at the top-right corner of this page.

Clone your fork

git clone https://github.com/devhmmza/connecdo-mvp.git
cd connecdo

Install dependencies

npm install

Create a new branch
Use a descriptive name for your branch:
git checkout -b feature/my-new-feature


Make your changes
Write code, add tests, or update docs.

Test your changes
Make sure everything works and tests pass.

Commit and push your branch: 
git add .
git commit -m "Add a helpful message describing your change"
git push origin feature/my-new-feature

Open a Pull Request (PR)
Go to your forked repo and open a PR to the main Connecdo repo.

Code Guidelines

Follow consistent code style and formatting.

Write clear, meaningful commit messages.

Add comments where needed to explain complex logic.

Write tests for new features or bug fixes.

Communication

Be respectful and kind.

Provide constructive feedback.

Ask questions if you’re unsure about something.

Thank you for helping make Connecdo better! 🎉
